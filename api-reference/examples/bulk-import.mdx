## Bulk Import Endpoints

Base URL: `https://api.casexchange.com/api/public/v1`

---

### POST /import/cases

Creates or upserts cases from a CSV file upload. The file must be a valid CSV with headers matching the expected case fields. Maximum file size is 20 MB.

**Tier:** `full`

```bash
curl -X POST "https://api.casexchange.com/api/public/v1/import/cases" \
  -H "X-API-Key: cxp_full_your_key_here" \
  -F "file=@cases.csv"
```

### POST /import/cases/update

Updates existing cases from a CSV file. Only existing cases are affected — no new records are created. Each row must include a valid case identifier so the system can match it to an existing record. Maximum file size is 20 MB.

> **Note:** The `notes` field has field-level RBAC. Only the receiving (referent) firm can update it. If the referring firm's CSV includes changes to `notes`, those rows will be rejected with a per-row error.

**Tier:** `full`

```bash
curl -X POST "https://api.casexchange.com/api/public/v1/import/cases/update" \
  -H "X-API-Key: cxp_full_your_key_here" \
  -F "file=@cases.csv"
```

**Response `200`**

```json

    "updated": 22,
    "skipped": 5,
    "errors": [
      {
        "row": 3,
        "field": "caseId",

        "message": "Permission denied. Only the receiving firm can update the 'notes' field."
      },
      {
```
