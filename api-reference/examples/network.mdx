## Network Endpoints

Base URL: `https://api.casexchange.com/api/public/v1`

---

### GET /network/outbound/{firmId}

Returns detailed information about a firm in your outbound network -- a firm you send referrals to. Includes the firm's profile, jurisdictions, case types, and the relationship settings you have configured for them.

**Tier:** `read_only`

```bash
curl -X GET "https://api.casexchange.com/api/public/v1/network/outbound/c3d4e5f6-7890-1abc-def2-34567890abcd" \
  -H "X-API-Key: cxp_ro_your_key_here"
```

**Response `200`**

```json
{
  "data": {
    "firmId": "c3d4e5f6-7890-1abc-def2-34567890abcd",
    "firmName": "Rivera Injury Law",

      "description": "Primary PI partner for Southern California referrals",

      "daysForUnderEvaluation": 14,
      "daysForInvestigating": 30,

    "requestId": "req_7f3a1b2c4d5e6f78",
```

### GET /network/pending

Returns all pending network relationship requests for your firm, including both inbound requests (firms asking to send you referrals) and outbound requests (firms you have requested to join your network that have not yet accepted).

**Tier:** `read_only`

```bash
curl -X GET "https://api.casexchange.com/api/public/v1/network/pending" \
  -H "X-API-Key: cxp_ro_your_key_here"
```

### POST /network/outbound/{firmId}

Creates a new outbound network relationship with the specified firm, or updates it if one already exists. Use this to request adding a firm to your referral network. The request body is optional -- omit it to create a relationship with default settings, or include fields to configure the relationship up front.

**Tier:** `standard`

```bash
curl -X POST "https://api.casexchange.com/api/public/v1/network/outbound/c3d4e5f6-7890-1abc-def2-34567890abcd" \
  -H "X-API-Key: cxp_std_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Strategic referral partner for CA personal injury",
    "sendingPercentage": 33.5,
    "daysForUnderEvaluation": 14,
    "daysForInvestigating": 30
  }'
```

**Response `200`**

```json
{
  "data": {
    "firmId": "c3d4e5f6-7890-1abc-def2-34567890abcd",
    "firmName": "Rivera Injury Law",

    "caseTypes": ["Personal Injury", "Product Liability"],
    "inNetwork": true,
    "generalReferralInstructions": "Email intake@riverainjury.example.com with case summary and client contact info.",
    "relationship": {
      "description": "Strategic referral partner for CA personal injury",
```

### PUT /network/outbound/{firmId}

Partially updates an existing outbound network relationship with the specified firm. Only the fields you include in the request body are changed; omitted fields remain as-is. Returns 404 if no relationship exists with the target firm.

**Tier:** `standard`

```bash
curl -X PUT "https://api.casexchange.com/api/public/v1/network/outbound/c3d4e5f6-7890-1abc-def2-34567890abcd" \
  -H "X-API-Key: cxp_std_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "sendingPercentage": 50,
    "referralInstructions": "Include police report and medical records when available"
  }'
```

**Response `200`**

```json
{
  "data": {
    "firmId": "c3d4e5f6-7890-1abc-def2-34567890abcd",
    "firmName": "Rivera Injury Law",
```

### PUT /network/inbound/{firmId}

Partially updates an existing inbound network relationship from the specified firm. Can be used to accept or reject a pending relationship request by setting the `status` field, or to update referral instructions. Returns 404 if no relationship exists from the specified firm.

**Tier:** `standard`

```bash
curl -X PUT "https://api.casexchange.com/api/public/v1/network/inbound/e5f6a7b8-9012-3cde-f456-7890abcdef01" \
  -H "X-API-Key: cxp_std_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "active",
    "referralInstructions": "Send cases via API with full intake details and signed retainer"
  }'
```

**Response `200`**

```json
{
  "data": {
    "firmId": "e5f6a7b8-9012-3cde-f456-7890abcdef01",
    "firmName": "Whitfield Trial Attorneys",

    "generalReferralInstructions": null,
    "relationship": {
      "description": null,
      "sendingPercentage": null,
      "daysForUnderEvaluation": null,
      "daysForInvestigating": null,
      "referralInstructions": "Send cases via API with full intake details and signed retainer"

  },

    "requestId": "req_2e8f6a7b9c012345",
```
