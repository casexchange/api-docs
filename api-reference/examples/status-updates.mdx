## Status Updates Endpoints

Base URL: `https://api.casexchange.com/api/public/v1`

---

### GET /status-updates/lookup

Returns all status updates and messages for a case or referral identified by a flexible identifier. The `id` query parameter accepts a referral UUID, a reference number (e.g. `CX-A1BC001-X2Y`), or a CMS ID from any connected system (DearLegal, Neos, Clio, SmartAdvocate, Salesforce, Case Status). When the identifier resolves to a base case, status updates across all accessible referrals are returned.

**Tier:** `read_only`

```bash
curl -X GET "https://api.casexchange.com/api/public/v1/status-updates/lookup?id=CX-A1BC001-X2Y" \
  -H "X-API-Key: cxp_ro_your_key_here"
```

### GET /status-updates

Returns all status updates across all cases and referrals the firm has access to, sorted by most recent first. Results are paginated.

Optional query parameters:

| Parameter  | Type     | Description                                                  |
|------------|----------|--------------------------------------------------------------|
| `page`     | integer  | Page number, 1-indexed (default `1`)                         |
| `limit`    | integer  | Results per page, 1-100 (default `20`)                       |
| `status`   | string   | Filter by status value (e.g. `signed`, `in_litigation`)      |
| `dateFrom` | ISO 8601 | Only status updates created on or after this date            |
| `dateTo`   | ISO 8601 | Only status updates created on or before this date           |
| `source`   | string   | Filter by source (`manual`, `case_status`, `salesforce`, `system`, `dearlegal`, `api`) |

**Tier:** `read_only`

```bash
curl -X GET "https://api.casexchange.com/api/public/v1/status-updates?page=1&limit=20&dateFrom=2026-03-01T00:00:00.000Z&dateTo=2026-03-31T23:59:59.000Z" \
  -H "X-API-Key: cxp_ro_your_key_here"
```
