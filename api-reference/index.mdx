### What is covered

- 58 paths and 74 operations in the OpenAPI schema (internal API).
- The Public API (`/api/public/v1`) adds dedicated endpoints for sent cases, received referrals, medical referrals, routing rules, notifications, and analytics — all authenticated via `X-API-Key`.
- Shared request/response schemas and standardized error envelopes across both API layers.

### Base URLs

| API | Base URL | Auth |
|-----|----------|------|
| **Public API** | `https://api.casexchange.com/api/public/v1` | `X-API-Key` header |
| **Internal API** | `https://api.casexchange.com/api/v1` | JWT Bearer token |

### Authentication

**Public API** — include your API key in every request:

```
X-API-Key: cxp_std_your_key_here
```

**Internal API** — use JWT bearer tokens:

1. `POST /auth/login`
2. `GET /auth/me`
3. `POST /auth/refresh-session` (when the access token expires)
4. Continue requests with `Authorization: Bearer <accessToken>`

See:

- `/api-reference/examples` for end-to-end request examples
- `/api-reference/error-handling` for auth failure troubleshooting

### Response conventions

Public API responses use a consistent envelope:

```json
{
  "data": { },
  "meta": { "requestId": "...", "timestamp": "..." },
  "pagination": {
    "total": 124,
    "page": 2,
    "limit": 20,
    "totalPages": 7
  }
}
```

Error responses:

```json
{
  "error": {
    "code": "validation_error",
    "message": "Request validation failed",
    "details": []
  },
  "meta": { "requestId": "...", "timestamp": "..." }
}
```
