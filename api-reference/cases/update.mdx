---
title: "Update Case"
description: "Update an existing case by ID"
---

<Endpoint method="PUT" path="/api/v1/cases/{id}" />

Update an existing case. Editable fields depend on the case status and your firm's relationship to the case. All fields are optional; only provided fields will be updated.

## Authentication

<ParamField header="Authorization" type="string" required>
Bearer token obtained from login or OAuth flow. Format: `Bearer <token>`
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
UUID of the case to update. Format: UUID v4.
</ParamField>

## Request Body

All fields are optional. Only provided fields will be updated.

<ParamField body="title" type="string">
Case title or summary. Maximum 200 characters.
</ParamField>

<ParamField body="description" type="string">
Detailed case description or summary. Maximum 5000 characters.
</ParamField>

<ParamField body="notes" type="string">
Internal notes or additional context. Maximum 2000 characters.
</ParamField>

<ParamField body="clientEmail" type="string">
Client's email address. Must be valid email format if provided.
</ParamField>

<ParamField body="clientPhone" type="string">
Client's phone number. Format: E.164 (e.g., `+14155551234`).
</ParamField>

<ParamField body="referentFirmId" type="string">
UUID of the firm receiving the referral. Use this to send a draft case or re-assign a case.
</ParamField>

## Permissions

Edit permissions depend on your firm's relationship to the case and the case status:

- **Referring firm**: Full edit before acknowledgment; limited fields (notes only) after acknowledgment.
- **Receiving firm**: Can edit notes once case is acknowledged.
- **System admin**: Unrestricted access to all fields.

<Warning>
Authorization derives from the JWT token. Ensure the user has access to the case's firm context. Attempting to update fields you don't have permission to modify will return a 403 Forbidden error.
</Warning>

## Example: Update Case with Error Handling

```javascript Node.js
async function updateCase(caseId, updates, accessToken) {
  try {
    const response = await fetch(`https://api.casexchange.com/api/v1/cases/${caseId}`, {
      method: 'PUT',
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(updates)
    });

    if (!response.ok) {
      const error = await response.json();
      
      if (response.status === 404) {
        throw new Error(`Case ${caseId} not found`);
      } else if (response.status === 403) {
        throw new Error(`Permission denied: ${error.error?.message}`);
      } else {
        throw new Error(error.error?.message || 'Failed to update case');
      }
    }

    const { data } = await response.json();
    return data;
  } catch (error) {
    console.error('Error updating case:', error.message);
    throw error;
  }
}

const updated = await updateCase(
  'c1d2e3f4-a5b6-7890-cdef-123456789012',
  {
    notes: 'Client has completed physical therapy. Updated on 2024-03-20.',
    clientEmail: 'maria.rodriguez.new@email.com'
  },
  accessToken
);

console.log(`Case ${updated.referenceNumber} updated at ${updated.updatedAt}`);
```
