---
title: "Re-Refer Case"
description: "Send a case to a new firm after rejection or routing changes"
---

<Endpoint method="POST" path="/api/v1/cases/{id}/re-refer" />

Re-refer a case to a different receiving firm. This creates a new referral record while preserving the original case history. Useful when a case is rejected, when routing rules change, or when you want to send the case to an alternative firm.

## Authentication

<ParamField header="Authorization" type="string" required>
Bearer token obtained from login or OAuth flow. Format: `Bearer <token>`
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
UUID of the base case (not a previous referral). Format: UUID v4.
</ParamField>

## Request Body

<ParamField body="referentFirmId" type="string" required>
UUID of the new firm receiving the referral. Use `/api/v1/firms` to discover valid firm IDs.
</ParamField>

<ParamField body="notes" type="string">
Optional notes explaining the re-referral reason. Maximum 2000 characters.
</ParamField>

<RequestExample>
```bash cURL
curl -X POST 'https://api.casexchange.com/api/v1/cases/c1d2e3f4-a5b6-7890-cdef-123456789012/re-refer' \
  -H 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...' \
  -H 'Content-Type: application/json' \
  -d '{
    "referentFirmId": "f3c4d5e6-f7g8-9012-cdef-12345678902",
    "notes": "Previous firm declined. Re-referring to alternative firm with expertise in this jurisdiction."
  }'
```
</RequestExample>

<RequestExample>

<ResponseField name="success" type="boolean" required>
Indicates whether the re-referral succeeded.
</ResponseField>

<ResponseField name="message" type="string" required>
Success message confirming the re-referral.
</ResponseField>

<ResponseField name="data" type="object" required>

<Expandable title="Referral properties">
<ResponseField name="id" type="string" required>
Unique identifier for the new referral. Format: UUID v4.
</ResponseField>

<ResponseField name="baseCaseId" type="string" required>
UUID of the original base case.
</ResponseField>

<ResponseField name="status" type="string" required>
Status of the new referral. Initially `sent`.
</ResponseField>

<ResponseField name="referringFirmId" type="string" required>
UUID of the firm creating the referral.

<ResponseField name="referentFirmId" type="string" required>
UUID of the firm receiving the referral.
</ResponseField>

## Business Rules

- **Ownership**: Only the case owner (referring firm) can re-refer cases.
- **Firm Relationship**: An active relationship must exist between your firm and the target firm.
- **History Preservation**: Creates a new referral record while preserving the original case and all prior referral history.
- **Audit Trail**: All referral attempts remain in the case timeline for auditing purposes.

<Warning>
Re-referring a case creates a new referral record. The original referral history is preserved, and the `sequenceNumber` increments to track multiple referral attempts.
</Warning>

## Example: Re-Refer Case with Error Handling

```javascript Node.js
async function reReferCase(caseId, newFirmId, notes, accessToken) {
  try {
    const response = await fetch(`https://api.casexchange.com/api/v1/cases/${caseId}/re-refer`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        referentFirmId: newFirmId,
        notes: notes
      })
    });

    if (!response.ok) {
      const error = await response.json();
      
      if (response.status === 404) {
        throw new Error(`Case or firm not found`);
      } else if (response.status === 403) {
        throw new Error('Only the case owner can re-refer cases');
      } else {
        throw new Error(error.error?.message || 'Failed to re-refer case');
      }
    }

    const { data } = await response.json();
    return data;
  } catch (error) {
    console.error('Error re-referring case:', error.message);
    throw error;
  }
}

const newReferral = await reReferCase(
  'c1d2e3f4-a5b6-7890-cdef-123456789012',
  'f3c4d5e6-f7g8-9012-cdef-12345678902',
  'Previous firm declined. Re-referring to alternative firm.',
  accessToken
);

console.log(`Case re-referred to new firm`);
console.log(`Sequence number: ${newReferral.sequenceNumber}`);
console.log(`New referral ID: ${newReferral.id}`);
```
