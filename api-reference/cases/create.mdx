---
title: "Create Case"
description: "Create a new case referral"
---

<Endpoint method="POST" path="/api/v1/cases" />

Create a new case referral. If `referentFirmId` is provided, the case is immediately sent to the receiving firm. Otherwise, it's created as a draft for later completion.

## Authentication

<ParamField header="Authorization" type="string" required>
Bearer token obtained from login or OAuth flow. Format: `Bearer <token>`
</ParamField>

## Request Body

<ParamField body="title" type="string" required>
Case title or summary. Maximum 200 characters.
</ParamField>

<ParamField body="caseType" type="string" required>
Type of case. Use `/api/v1/firms/case-types` to get valid values. Examples: `personal_injury`, `medical_malpractice`, `product_liability`.
</ParamField>

<ParamField body="jurisdiction" type="string" required>
Jurisdiction code (e.g., `CA`, `NY`, `TX`). Use `/api/v1/firms/jurisdictions/list` for valid codes.
</ParamField>

<ParamField body="clientFirstName" type="string" required>
Client's first name.
</ParamField>

<ParamField body="clientLastName" type="string" required>
Client's last name.
</ParamField>

<ParamField body="description" type="string">
Detailed case description or summary. Maximum 5000 characters.
</ParamField>

<ParamField body="clientEmail" type="string">
Client's email address. Must be valid email format if provided.
</ParamField>

<ParamField body="clientPhone" type="string">
Client's phone number. Format: E.164 (e.g., `+14155551234`).
</ParamField>

<ParamField body="referentFirmId" type="string">
UUID of the firm receiving the referral. If omitted, case is created as `draft`. Use `/api/v1/firms` to discover valid firm IDs.
</ParamField>

<ParamField body="notes" type="string">
Internal notes or additional context. Maximum 2000 characters.
</ParamField>

<ParamField body="salesforceRecordId" type="string">
Salesforce record ID for CRM integration. Must be provided together with `salesforceObjectType`.
</ParamField>

<ParamField body="salesforceObjectType" type="string">
Salesforce object type (e.g., `Case`, `Opportunity`). Must be provided together with `salesforceRecordId`.
</ParamField>

## Example: Create Draft Case and Send Later

```javascript Node.js
async function createDraftCase(caseData) {
  const draftCase = {
    ...caseData,
    referentFirmId: undefined
  };

  const response = await fetch('https://api.casexchange.com/api/v1/cases', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${accessToken}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(draftCase)
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.error?.message || 'Failed to create case');
  }

  return await response.json();
}

async function sendCase(caseId, referentFirmId) {
  const response = await fetch(`https://api.casexchange.com/api/v1/cases/${caseId}`, {
    method: 'PUT',
    headers: {
      'Authorization': `Bearer ${accessToken}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ referentFirmId })
  });

  return await response.json();
}

const draft = await createDraftCase({
  title: 'Motor Vehicle Accident - Rear End Collision',
  caseType: 'personal_injury',
  jurisdiction: 'CA',
  clientFirstName: 'Maria',
  clientLastName: 'Rodriguez'
});

console.log(`Draft created: ${draft.data.referenceNumber}`);

const sent = await sendCase(draft.data.id, 'f2b3c4d5-e6f7-8901-bcde-f12345678901');
console.log(`Case sent: ${sent.data.status}`);
```

<Note>
Status defaults to `draft` unless `referentFirmId` is supplied. Use `/api/v1/firms` to discover valid `referentFirmId` values before creating cases.
</Note>

<Warning>
Salesforce fields (`salesforceRecordId` and `salesforceObjectType`) must be provided together. If one is included without the other, the request will fail with a validation error.
</Warning>
