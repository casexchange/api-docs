<Endpoint method="GET" path="/api/v1/cases" />

Retrieve a paginated list of cases with optional filtering. Supports filtering by status, case type, jurisdiction, date range, and search terms. Results are scoped to cases visible to the authenticated user's firm.

## Authentication

<ParamField header="Authorization" type="string" required>
Bearer token obtained from login or OAuth flow. Format: `Bearer <token>`
</ParamField>

## Query Parameters

<ParamField query="page" type="integer" default="1">
Page number for pagination. Minimum: 1.
</ParamField>

<ParamField query="limit" type="integer" default="20">
Number of results per page. Range: 1-100. Default: 20.
</ParamField>

<ParamField query="search" type="string">
Search term to match against case title or client name (first name, last name, or full name). Case-insensitive partial matching.
</ParamField>

<ParamField query="status" type="string">
Comma-separated list of status values to filter by. Valid values: `draft`, `sent`, `under_evaluation`, `investigating`, `signed`, `closed`, `rejected`. Example: `under_evaluation,investigating`.
</ParamField>

<ParamField query="caseType" type="string">
Filter by case type. Use `/api/v1/firms/case-types` to get valid values. Example: `personal_injury`.
</ParamField>

<ParamField query="jurisdiction" type="string">
Filter by jurisdiction code. Use `/api/v1/firms/jurisdictions/list` for valid codes. Example: `CA`.
</ParamField>

<ParamField query="view" type="string" default="all">
Filter by case direction. Values: `sent` (cases your firm originated), `received` (inbound referrals), `all` (both sent and received).
</ParamField>

<ParamField query="fromDate" type="string">
Start date for filtering cases by creation date. Format: ISO 8601 (e.g., `2024-03-01T00:00:00Z`).
</ParamField>

<ParamField query="toDate" type="string">
End date for filtering cases by creation date. Format: ISO 8601 (e.g., `2024-03-31T23:59:59Z`).
</ParamField>

## Example: Paginated Case Listing with Filters

```javascript Node.js
async function listCases(filters = {}, accessToken) {
  const params = new URLSearchParams({
    page: filters.page?.toString() || '1',
    limit: filters.limit?.toString() || '20',
    view: filters.view || 'all'
  });

  if (filters.search) {
    params.append('search', filters.search);
  }

  if (filters.status) {
    params.append('status', Array.isArray(filters.status) ? filters.status.join(',') : filters.status);
  }

  if (filters.caseType) {
    params.append('caseType', filters.caseType);
  }

  if (filters.jurisdiction) {
    params.append('jurisdiction', filters.jurisdiction);
  }

  if (filters.fromDate) {
    params.append('fromDate', filters.fromDate);
  }

  if (filters.toDate) {
    params.append('toDate', filters.toDate);
  }

  const response = await fetch(`https://api.casexchange.com/api/v1/cases?${params}`, {
    headers: {
      'Authorization': `Bearer ${accessToken}`
    }
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.error?.message || 'Failed to list cases');
  }

  return await response.json();
}

async function getAllCases(filters = {}, accessToken) {
  const allCases = [];
  let page = 1;
  let hasMore = true;

  while (hasMore) {
    const result = await listCases({ ...filters, page }, accessToken);
    allCases.push(...result.data);

    hasMore = page < result.pagination.totalPages;
    page++;
  }

  return allCases;
}

const cases = await listCases({
  status: ['under_evaluation', 'investigating'],
  view: 'sent',
  caseType: 'personal_injury',
  jurisdiction: 'CA'
}, accessToken);

console.log(`Found ${cases.pagination.total} cases`);
cases.data.forEach(caseItem => {
  console.log(`${caseItem.referenceNumber}: ${caseItem.title} (${caseItem.status})`);
});
```

<Tip>
Use `view=sent` to see cases your firm has originated, `view=received` for inbound referrals. Apply `fromDate` / `toDate` to limit large result sets during nightly syncs or when exporting historical data.
</Tip>

<Note>
Results are scoped to cases visible to the authenticated user's firm. The `view` parameter controls whether you see sent cases, received cases, or both.
</Note>
