<Endpoint method="GET" path="/api/v1/cases/{id}" />

Retrieve detailed information about a specific case by its UUID. Includes full case details, firm associations, status history, and document references.

## Authentication

<ParamField header="Authorization" type="string" required>
Bearer token obtained from login or OAuth flow. Format: `Bearer <token>`
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
UUID of the case to retrieve. Format: UUID v4.
</ParamField>

<RequestExample>
```bash cURL
curl -X GET 'https://api.casexchange.com/api/v1/cases/c1d2e3f4-a5b6-7890-cdef-123456789012' \
  -H 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
```
</RequestExample>

<RequestExample>
```javascript Node.js
const caseId = 'c1d2e3f4-a5b6-7890-cdef-123456789012';

const response = await fetch(`https://api.casexchange.com/api/v1/cases/${caseId}`, {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});

const data = await response.json();
console.log('Case:', data.data.referenceNumber, data.data.status);
```
</RequestExample>

## Example: Retrieve Case with Error Handling

```javascript Node.js
async function getCase(caseId, accessToken) {
  try {
    const response = await fetch(`https://api.casexchange.com/api/v1/cases/${caseId}`, {
      headers: {
        'Authorization': `Bearer ${accessToken}`
      }
    });

    if (!response.ok) {
      const error = await response.json();
      
      if (response.status === 404) {
        throw new Error(`Case ${caseId} not found`);
      } else if (response.status === 403) {
        throw new Error('You do not have permission to access this case');
      } else {
        throw new Error(error.error?.message || 'Failed to retrieve case');
      }
    }

    const { data } = await response.json();
    
    return {
      id: data.id,
      referenceNumber: data.referenceNumber,
      title: data.title,
      status: data.status,
      clientName: `${data.clientFirstName} ${data.clientLastName}`,
      referringFirm: data.referringFirm.name,
      referentFirm: data.referentFirm?.name,
      statusHistory: data.statusUpdates.length,
      documentCount: data.documents.length,
      createdAt: data.createdAt
    };
  } catch (error) {
    console.error('Error retrieving case:', error.message);
    throw error;
  }
}

const caseInfo = await getCase(
  'c1d2e3f4-a5b6-7890-cdef-123456789012',
  accessToken
);

console.log(`Case ${caseInfo.referenceNumber}: ${caseInfo.title}`);
console.log(`Status: ${caseInfo.status}`);
console.log(`Status updates: ${caseInfo.statusHistory}`);
console.log(`Documents: ${caseInfo.documentCount}`);
```

<Note>
Includes nested firm details, status history, and document references. The `statusUpdates` and `documents` arrays may be empty for new cases or large for cases with extensive history. Consider paginating these arrays on the client when rendering.
</Note>

<Warning>
Permissions mirror the caller's firm context. Only cases visible to the authenticated user's firm can be retrieved. Attempting to access cases from other firms will return a 403 Forbidden error.
</Warning>
